Var : The scope of a var variable is functional scope.It can be updated and re-declared into the scope.It can be declared without initialization.It can be accessed without initialization as its default value is “undefined”.hoisting done , with initializing as ‘default’ value.

let : The scope of a let variable is block scope.It can be updated but cannot be re-declared into the scope.It can be declared without initialization.It cannot be accessed without initialization otherwise it will give ‘referenceError’.Hoisting is done , but not initialized (this is the reason for error when we access the let variable before declaration/initialization.

No this : In a traditional function, its internal 'this' value is dynamic, it depends on how the function is invoked. In the arrow function, there are no this, if we access this in the arrow function it will return. the this of the closest non-arrow parent function.

No arguments : In a traditional function, there is an arguments local variable. If the number of arguments to our function is dynamic, using arguments allows us to easily do things like calculating the maximum number of passed arguments.In the arrow function, there are no arguments, if we access arguments in the arrow function will return the arguments of the closest non-arrow parent function.

Duplicate named parameters are not allowed : In non-restrict mode, traditional functions allow us to use duplicate named parameters. But in strict mode, it is not allowed.In arrow functions, parameters with the same name are not allowed whether the strict mode is enabled or not.

No prototype : We can get prototype for traditional function, but the arrow function does not have prototype .

Cannot be invoked with new : We can use the new keyword on the traditional function to create a new object.But arrow functions cannot be called with new.

What even are JavaScript Array Methods? : .forEach(), .map(), .filter() and .find(). When you go to loop through (or iterate through) an array, the first thing you think of is probably a for loop. .forEach(), .map(), .filter() and .find() are all just other ways of iterating through arrays to perform a specific task on each element of the array, and are called methods.

.forEach(), What is it? : .forEach() is a very generic array method. We should try to only use it when we want to perform a specific action for each element of an array.

.map(), What is it? : .map() is a lot like .forEach(), but it very helpfully creates and returns a new array.Use .map() whenever you need to update data inside an array (by mapping over it!)..map() will always return a new array of the same length as the original!

.filter(), What is it? : .filter() loops through (or iterates) through data, and filters out data that doesn't match the criteria that we set. We define what those criteria are through a truth test inside a function.

.find(), What is it? : .filter() loops through (or iterates) through data, and filters out the first data that doesn't match the criteria that we set. We define what those criteria are through a truth test inside a function.
